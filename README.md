# secure_copy — многопоточное копирование с шифрованием

## Описание

Программа `secure_copy` предназначена для параллельного шифрования (или дешифрования) нескольких файлов с использованием простого XOR-алгоритма (библиотека `libcaesar.so` из предыдущего задания). Она демонстрирует принципы многопоточного программирования в POSIX-системах: создание и управление потоками, синхронизацию доступа к общим ресурсам с помощью мьютексов, обработку взаимоблокировок с использованием таймаутов, а также обработку сигналов (SIGINT) для корректного завершения.

Программа всегда запускает ровно три рабочих потока. Каждый поток обрабатывает один файл целиком (чтение, шифрование, запись). Если файлов меньше трёх, часть потоков простаивает. Если файлов больше трёх, они помещаются в очередь и распределяются между потоками по мере освобождения.

После успешной обработки каждого файла в файл `log.txt` добавляется запись с временем, идентификатором процесса (PID), идентификатором потока (TID) и именем обработанного файла.

При получении сигнала `SIGINT` (Ctrl+C) программа корректно завершает работу: потоки прерывают обработку текущего файла, закрываются все открытые файлы и выводится сообщение `"Операция прервана пользователем"`.

## Требования

- Операционная система Linux (или WSL2 под Windows 11).
- Установленная библиотека `libcaesar.so` (из Задания 1). Библиотека должна находиться либо в текущей папке, либо в системном пути (например, `/usr/local/lib`).
- Компилятор `g++` с поддержкой C++11.
- Утилита `make`.
- Библиотека `pthread` (обычно входит в состав `build-essential`).

## Сборка

1. Поместите файлы `secure_copy.cpp`, `Makefile` и `libcaesar.so` в одну директорию.
2. Выполните команду:
   ```bash
   make secure_copy
   (или просто make, если цель all настроена на сборку secure_copy).

В результате будет создан исполняемый файл secure_copy.

Использование
Синтаксис командной строки:

bash
./secure_copy <файл1> [файл2] ... [файлN] <выходная_папка> <ключ>
<файл1> ... <файлN> — имена входных файлов (минимум один). Если файлов нет, программа выведет ошибку.

<выходная_папка> — имя папки, в которую будут помещены зашифрованные файлы. Если папка не существует, она будет создана.

<ключ> — целое число от 0 до 255, используемое для XOR-шифрования.

Примеры:

bash
# Шифрование трёх файлов
./secure_copy a.txt b.txt c.txt out 123

# Шифрование одного файла
./secure_copy document.txt secret 42

# Дешифрование (применяется тот же ключ)
./secure_copy secret/document.txt restored 42
После выполнения в папке out (или указанной вами) появятся файлы с теми же именами, но с зашифрованным содержимым. В текущей директории будет создан (или дополнен) файл log.txt с записями об обработке.

Логирование
Файл log.txt имеет следующий формат строки:

text
YYYY-MM-DD HH:MM:SS PID:12345 TID:12345678 filename.txt
Дата и время окончания обработки файла.

PID процесса.

TID потока (натуральный идентификатор, возвращаемый pthread_self()).

Имя обработанного файла (только имя, без пути).

Обработка сигналов
При нажатии Ctrl+C программа перехватывает сигнал SIGINT, устанавливает глобальный флаг завершения. Потоки, проверяющие этот флаг в циклах, прерывают обработку текущего файла и завершаются. Главный поток ожидает завершения всех рабочих потоков, выводит сообщение "Операция прервана пользователем" и завершает программу. Частично записанные выходные файлы не удаляются (при желании их можно удалить вручную).

Механизмы синхронизации и предотвращение взаимоблокировок
Мьютекс очереди: Доступ к очереди файлов (когда число файлов > 3) защищён мьютексом.

Таймаут: При попытке захвата мьютекса используется pthread_mutex_timedlock с таймаутом 5 секунд. Если мьютекс не удаётся захватить за это время, программа выводит сообщение об ошибке ("Cannot acquire queue mutex within 5 seconds – possible deadlock") и аварийно завершается с кодом 1.

Условные переменные: Для ожидания появления новых задач в очереди используются условные переменные, что позволяет избежать активного ожидания.

Тестирование
Для проверки работоспособности прилагается Makefile с целями:

make testfile — создаёт тестовый файл размером 10 МБ (test10M.bin) со случайными данными.

make test_secure — выполняет шифрование и дешифрование этого файла с ключом 123 и сравнивает результат.

make test_interrupt — запускает программу для проверки обработки Ctrl+C (требуется ручное нажатие).

Также можно протестировать на нескольких файлах:

bash
# Подготовка
echo "Hello" > file1.txt
echo "World" > file2.txt
mkdir out

# Шифрование
./secure_copy file1.txt file2.txt out 99

# Проверка (расшифровка)
./secure_copy out/file1.txt out/file2.txt dec 99
cmp file1.txt dec/file1.txt
cmp file2.txt dec/file2.txt
Структура проекта
secure_copy.cpp — исходный код программы.

Makefile — файл для автоматизации сборки и тестирования.

libcaesar.so — динамическая библиотека с функциями set_key и caesar (должна быть предоставлена отдельно).

README.md — данный файл.

Замечания
Библиотека libcaesar.so должна быть скомпилирована с флагами -fPIC и содержать функции с extern "C". Если библиотека находится не в текущей папке, укажите путь при линковке или установите её в систему (sudo make install из Задания 1).

Для корректной работы с таймаутами может потребоваться линковка с -lrt (в современных glibc это необязательно, но в Makefile флаг присутствует для совместимости).

При компиляции используются флаги -Wall -pthread.
